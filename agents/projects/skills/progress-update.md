# SKILL: progress-update

## Trigger
Update, log progress, worked on, shipped, finished, done with

## Steps
1. Identify the single project the update belongs to — match against the projects list
2. Compress the update into one past-tense line capturing the outcome, not the effort
3. Record it and confirm; mention the gap if the project had been quiet a while

## Output Format
- One confirmation line: what was logged and for which project
- No follow-up questions unless the project is genuinely ambiguous

## Edge Cases
- Update mentions two projects: log the primary one and say so
- Update matches an on-hold project: log it and ask if the project is active again
